## 1. التثبيت

### Docker Compose (الطريقة الموصى بها)

أبسط طريقة لتشغيل فنك. يتولى Docker إعداد قاعدة البيانات والمفاتيح السرية ومخطط قاعدة البيانات تلقائياً.

1. استنسخ المستودع:
   ```bash
   git clone https://github.com/mulaifi/fanek.git
   cd fanek
   ```
2. شغّل المكدّس:
   ```bash
   docker compose up -d
   ```
3. افتح http://localhost:3000 واتبع معالج الإعداد.

هذا كل شيء. تقوم نقطة دخول Docker تلقائياً بما يلي:
- توليد `NEXTAUTH_SECRET` آمن عند التشغيل الأول (يُحفظ عبر عمليات إعادة التشغيل)
- انتظار جاهزية PostgreSQL
- تطبيق ترحيلات قاعدة البيانات

التخصيص: لتجاوز الإعدادات الافتراضية، أنشئ ملف `.env` في جذر المشروع قبل التشغيل. يلتقطه Docker Compose تلقائياً. راجع ملف README للاطلاع على جميع متغيرات البيئة.

ملاحظات الإنتاج:
- اضبط `NEXTAUTH_URL` على نطاقك الفعلي (مثلاً `https://fanek.example.com`) في ملف `.env`
- يُنصح باستخدام نسخة PostgreSQL خارجية مع نسخ احتياطية مناسبة
- غيّر بيانات اعتماد قاعدة البيانات الافتراضية في `docker-compose.yml` أو عبر ملف `.env`

### التثبيت اليدوي

المتطلبات: Node.js 20 أو أحدث، PostgreSQL 16 أو أحدث.

1. استنسخ المستودع وثبّت التبعيات:
   ```bash
   git clone https://github.com/mulaifi/fanek.git
   cd fanek
   npm install
   ```
2. أنشئ قاعدة البيانات:
   ```bash
   createdb fanek
   ```
3. هيّئ متغيرات البيئة:
   ```bash
   cp .env.example .env
   ```
   افتح `.env` واضبط:
   - `DATABASE_URL` -- سلسلة اتصال PostgreSQL
   - `NEXTAUTH_SECRET` -- ولّد بالأمر `openssl rand -hex 32`
   - `NEXTAUTH_URL` -- الرابط الذي سيُوصل منه إلى فنك
4. طبّق مخطط قاعدة البيانات:
   ```bash
   npx prisma migrate deploy
   ```
5. شغّل الخادم:
   ```bash
   npm run build && npm start
   ```
   للتطوير: `npm run dev`

افتح http://localhost:3000 واتبع معالج الإعداد.


## 2. معالج الإعداد

يعمل معالج الإعداد تلقائياً عند أول استخدام. إذا لم يكن هناك سجل إعداد مكتمل في قاعدة البيانات، فإن أي زيارة للتطبيق تعيد التوجيه إلى `/setup`. بعد اكتمال الإعداد، يُعطّل المعالج نهائياً ولا يمكن تشغيله مرة أخرى.

يتكون المعالج من أربع خطوات.

### الخطوة 1 -- حساب المسؤول

أدخل اسمك وبريدك الإلكتروني وكلمة مرور لحساب المسؤول الأولي. يُقيّم مقياس قوة كلمة المرور كلمتك وفق خمسة معايير: الحد الأدنى للطول، الأحرف الصغيرة، الأحرف الكبيرة، الأرقام، والرموز الخاصة. استهدف "قوية" أو "قوية جداً" قبل المتابعة.

هذا الحساب سيحصل على دور المسؤول مع صلاحيات كاملة. يمكنك إضافة مسؤولين آخرين لاحقاً.

### الخطوة 2 -- معلومات المؤسسة

أدخل اسم مؤسستك. يظهر هذا الاسم في الشريط الجانبي وفي البيانات المُصدّرة. يمكنك أيضاً رفع شعار هنا (PNG أو JPEG أو GIF أو WebP، بحد أقصى 256 كيلوبايت). يمكن تغيير كليهما لاحقاً من الإعدادات.

### الخطوة 3 -- قالب البداية

اختر القالب الأنسب لنشاطك التجاري. تملأ القوالب كتالوج الخدمات مسبقاً بأنواع الخدمات وهياكل الحقول حتى لا تبدأ من صفحة فارغة.

| القالب | المحتوى المُعبّأ مسبقاً |
|--------|------------------------|
| مزود خدمات سحابية | مركز بيانات افتراضي، استعادة الكوارث، النسخ الاحتياطي، والاتصال |
| اتصالات | صوت، بيانات، إنترنت، وخدمات مُدارة |
| مزود خدمات مُدارة | تقنية معلومات مُدارة، أمن، نسخ احتياطي، ودعم فني |
| فارغ | بدون أنواع خدمات، ابنِ الكتالوج بنفسك |

يمكنك تعديل أو إضافة أو إزالة أي نوع خدمة بعد الإعداد. القالب يحدد نقطة البداية فقط.

### الخطوة 4 -- الاكتمال

يحفظ فنك الإعدادات ويعيد توجيهك إلى لوحة المعلومات بعد تأخير قصير. سجّل الدخول باستخدام بيانات اعتماد المسؤول التي أنشأتها في الخطوة 1.


## 3. إدارة المستخدمين

انتقل إلى **الإدارة > المستخدمون** لإدارة جميع حسابات المستخدمين. يمكن للمسؤولين فقط الوصول إلى هذه الصفحة.

### دعوة مستخدم جديد

1. انقر **دعوة مستخدم**.
2. أدخل الاسم الكامل للمستخدم وبريده الإلكتروني وحدد دوراً (المشاهد هو الافتراضي).
3. انقر **إرسال الدعوة**.

يُنشئ فنك الحساب فوراً ويعرض كلمة مرور مؤقتة في شريط أعلى الصفحة. **مهم:** انسخ كلمة المرور فوراً قبل إغلاق الشريط، إذ لا يمكن استرجاعها لاحقاً. شارك كلمة المرور مع المستخدم عبر قناة آمنة مثل تطبيق مراسلة مشفّر من طرف إلى طرف أو أداة مخصصة لمشاركة كلمات المرور. تجنب إرسال كلمات المرور عبر البريد الإلكتروني أو المحادثات غير المشفرة. يُغلق الشريط عند النقر على "إغلاق" ولا يظل ظاهراً. لا يوجد إرسال بريد إلكتروني مدمج.

الدور الافتراضي للدعوات الجديدة هو المشاهد. غيّر الدور في نموذج الدعوة إذا كنت بحاجة لمنح صلاحيات أكبر.

### تغيير دور المستخدم

1. ابحث عن المستخدم في الجدول وانقر **تعديل الدور** في عمود الإجراءات.
2. اختر الدور الجديد من القائمة المنسدلة (مشاهد أو محرر أو مسؤول).
3. انقر **حفظ**.

يسري تغيير الدور فوراً. سيخضع الإجراء التالي للمستخدم للصلاحيات الجديدة. للاطلاع على صلاحيات كل دور، راجع [الأدوار والصلاحيات](#8-الأدوار-والصلاحيات).

### إعادة تعيين كلمة المرور

إذا فقد المستخدم الوصول أو نسي كلمة مروره، انقر **إعادة تعيين كلمة المرور** في صف المستخدم. يولّد فنك كلمة مرور مؤقتة عشوائية جديدة ويعرضها في شريط. **مهم:** انسخ كلمة المرور فوراً قبل إغلاق الشريط. شاركها مع المستخدم عبر قناة آمنة (مراسلة مشفّرة من طرف إلى طرف أو أداة مشاركة كلمات مرور). عند تسجيل الدخول بكلمة المرور المؤقتة، سيُطلب منه تعيين كلمة مرور جديدة قبل الوصول إلى لوحة المعلومات.

### إلغاء الجلسات

انقر **إلغاء الجلسات** لإبطال جميع الجلسات النشطة لذلك المستخدم فوراً. هذا مفيد عند فقدان جهاز أو اختراق حساب أو مغادرة موظف. سيتم تسجيل خروج المستخدم من جميع الأجهزة ويجب عليه تسجيل الدخول مرة أخرى.

### حذف مستخدم

انقر **حذف** في صف المستخدم. يتطلب الزر نقرة ثانية للتأكيد (يظهر زر "تأكيد؟" مع عد تنازلي لمدة 5 ثوانٍ. إذا لم تؤكد خلال 5 ثوانٍ، يُلغى الإجراء). لا يمكنك حذف حسابك الخاص.

يفقد المستخدمون المحذوفون الوصول فوراً. تُحفظ بياناتهم التاريخية (العملاء، الخدمات، سجلات المراجعة).

### تدفق تسجيل الدخول الأول

عندما يسجّل المستخدم الدخول بكلمة مرور مؤقتة (سواء من دعوة أو إعادة تعيين)، يفرض فنك تغيير كلمة المرور قبل الوصول إلى أي صفحة أخرى. بعد تعيين كلمة مرور جديدة، ينتقل إلى لوحة المعلومات بشكل طبيعي.


## 4. كتالوج الخدمات

انتقل إلى **الإدارة > كتالوج الخدمات** لإدارة أنواع الخدمات. كتالوج الخدمات هو أقوى ميزة تهيئة في فنك. يحدد بنية الخدمات التي تتتبعها لعملائك.

### ما هو نوع الخدمة؟

نوع الخدمة هو قالب. يحتوي على اسم ووصف اختياري ورمز تعبيري اختياري وهيكل حقول، وهو قائمة بالحقول المخصصة التي ستكون لكل خدمة من هذا النوع. عندما ينشئ محرر خدمة لعميل ويختار نوعاً، يعرض فنك نموذجاً ديناميكياً مبنياً من هيكل حقول ذلك النوع.

على سبيل المثال، قد يحتوي نوع خدمة "مركز بيانات افتراضي" على حقول لـ "عدد المعالجات الافتراضية" و"الذاكرة (جيجابايت)" و"التخزين (تيرابايت)" و"انتهاء العقد". وقد يحتوي نوع خدمة "خط مؤجّر" على حقول لـ "النطاق الترددي (ميجابت/ثانية)" و"CIR" و"مرجع المزود".

### إنشاء نوع خدمة

1. انقر **نوع خدمة جديد**.
2. أدخل اسماً (مطلوب). يظهر الاسم في قائمة أنواع الخدمات وفي نماذج خدمات العملاء.
3. أضف وصفاً اختيارياً لتوضيح الغرض من النوع للمحررين الآخرين.
4. أدخل رمزاً تعبيرياً اختيارياً في حقل الأيقونة (مثلاً `☁️`). يظهر الرمز بجانب الاسم في القوائم.
5. اترك مفتاح **نشط** مفعّلاً. الأنواع غير النشطة مخفية عن المحررين عند إنشاء خدمات جديدة.
6. أضف الحقول باستخدام منشئ هيكل الحقول (انظر أدناه).
7. انقر **إنشاء نوع الخدمة**.

### بناء هيكل الحقول

يحدد هيكل الحقول حقول البيانات التي تظهر في كل خدمة من هذا النوع. استخدم المنشئ لإضافة الحقول وتهيئتها وإعادة ترتيبها وإزالتها.

لكل حقل، اضبط:

- **اسم الحقل** -- معرّف يُقرأ آلياً، بأحرف صغيرة وشرطات سفلية (مثلاً `contract_value`). تُحوّل المسافات تلقائياً إلى شرطات سفلية.
- **التسمية المعروضة** -- التسمية المقروءة التي تظهر في النموذج (مثلاً `قيمة العقد`).
- **النوع** -- أحد الأنواع التالية:
  - **نص** -- حقل إدخال نصي أحادي السطر
  - **رقم** -- حقل إدخال رقمي
  - **تاريخ** -- منتقي تاريخ
  - **قائمة منسدلة** -- قائمة منسدلة بخيارات محددة مسبقاً تحددها أنت
  - **عملة** -- حقل إدخال رقمي للمبالغ المالية
  - **مربع اختيار (نعم/لا)** -- مفتاح تبديل منطقي
- **مطلوب** -- فعّل هذا إذا كان يجب ملء الحقل قبل حفظ الخدمة.

لحقول **القائمة المنسدلة**، أدخل الخيارات المسموحة كقائمة مفصولة بفواصل (مثلاً `نشط, معلّق, ملغى`). يحلل فنك القائمة أثناء الكتابة ويعرض شارات الخيارات أسفل حقل الإدخال.

استخدم أسهم الأعلى والأسفل في كل بطاقة حقل لإعادة الترتيب. الترتيب هنا هو الترتيب الذي تظهر به في نموذج الخدمة. انقر أيقونة الحذف لإزالة حقل.

### تعديل نوع خدمة موجود

انقر **تعديل** في أي صف (أو انقر الصف نفسه) لفتح المحرر المضمّن. يمكن تغيير جميع الخصائص بما في ذلك هيكل الحقول.

**التأثير على الخدمات الموجودة:** يخزّن فنك قيم حقول الخدمة كمستند JSON مرن. إضافة حقول جديدة للهيكل ستجعلها تظهر فارغة في الخدمات الموجودة. إزالة حقل من الهيكل تخفيه من النموذج، لكن البيانات المخزنة لا تُحذف. إعادة تسمية الاسم الداخلي لحقل تقطع الرابط مع القيم المخزنة لذلك الحقل في الخدمات الموجودة، فتصبح البيانات القديمة يتيمة. أعد تسمية التسميات المعروضة بحرية، لكن أعد تسمية أسماء الحقول بحذر.

### إلغاء تنشيط نوع خدمة

أوقف مفتاح **نشط** في نموذج نوع الخدمة واحفظ. تُزال الأنواع غير النشطة من قائمة "إنشاء خدمة" المنسدلة للمحررين، فلا يمكن إنشاء خدمات جديدة من هذا النوع. لا تتأثر الخدمات الموجودة من هذا النوع وتبقى مرئية.

### حذف نوع خدمة

يكون زر الحذف معطّلاً إذا كانت هناك خدمات من هذا النوع. لحذف نوع، احذف أو أعد تعيين جميع الخدمات التي تستخدمه أولاً. الحذف إجراء بنقرتين مع عد تنازلي لمدة 5 ثوانٍ.


## 5. الإعدادات

انتقل إلى **الإدارة > الإعدادات** لتهيئة الإعدادات على مستوى التطبيق. تحتوي صفحة الإعدادات على أربع علامات تبويب.

### علامة تبويب المؤسسة

- **اسم المؤسسة** -- الاسم المعروض في التطبيق وفي التصديرات. مطلوب.
- **شعار المؤسسة** -- انقر مربع الشعار أو زر الرفع لاختيار ملف صورة (PNG أو JPEG أو GIF أو WebP، بحد أقصى 256 كيلوبايت). تُخزّن الصورة في قاعدة البيانات بصيغة base64 data URL. انقر **إزالة** للعودة إلى عنصر الحرف الأول البديل.
- **اللغة الافتراضية** -- تحدد اللغة الافتراضية للمستخدمين الجدد. تدعم حالياً الإنجليزية والعربية. يمكن للمستخدمين تبديل اللغة من شريط التنقل بشكل مستقل عن هذا الإعداد.

انقر **حفظ** لتطبيق التغييرات.

### علامة تبويب المصادقة

تهيئة موفّري تسجيل الدخول عبر OAuth. يدعم فنك حالياً Google OAuth.

لتفعيل تسجيل الدخول عبر Google:

1. أنشئ عميل OAuth 2.0 في [Google Cloud Console](https://console.cloud.google.com/).
2. اضبط URI إعادة التوجيه المصرّح به على `https://your-domain/api/auth/callback/google`.
3. انسخ معرّف العميل والمفتاح السري في الحقول المعنية.
4. انقر **حفظ**.

سيظهر خيار تسجيل الدخول عبر Google في صفحة تسجيل الدخول فوراً. إذا كان معرّف العميل أو المفتاح السري فارغاً، يُخفى خيار Google.

يُشفّر المفتاح السري للعميل قبل تخزينه في قاعدة البيانات. لا يُعاد أبداً إلى المتصفح بنص واضح.

### علامة تبويب الحالات

حالات العملاء هي التسميات المستخدمة لتصنيف عملائك (مثلاً نشط، غير نشط، محتمل، منسحب). قام قالب البداية الذي اخترته أثناء الإعداد بتعبئة قائمة أولية.

من علامة التبويب هذه يمكنك:

- **إضافة حالة** -- اكتب تسمية جديدة واضغط زر الإضافة.
- **تعديل حالة** -- انقر أيقونة التعديل بجانب أي حالة وحدّث الاسم.
- **إعادة ترتيب الحالات** -- اسحب الحالات لأعلى ولأسفل للتحكم في ترتيب ظهورها في المرشحات والقوائم المنسدلة.
- **حذف حالة** -- انقر أيقونة الحذف. لاحظ أن حذف حالة لا يحدّث تلقائياً العملاء الذين لديهم تلك الحالة.

تسري التغييرات فوراً بعد الحفظ.

### علامة تبويب البيانات

انقر **تصدير جميع البيانات** لتنزيل تصدير JSON كامل لبيانات فنك، شاملاً العملاء والشركاء والخدمات وأنواع الخدمات. يُسمّى الملف `fanek-export.json`.

استخدم هذا للنسخ الاحتياطي أو الترحيل أو التكامل مع أنظمة أخرى.


## 6. سجل المراجعة

انتقل إلى **الإدارة > سجل المراجعة** لعرض سجل زمني لجميع عمليات تغيير البيانات في فنك. هذه الصفحة متاحة للمسؤولين فقط.

تُسجّل تلقائياً كل عملية إنشاء وتحديث وحذف على العملاء والشركاء والخدمات والمستخدمين والإعدادات وأنواع الخدمات مع طابع زمني واسم المستخدم المنفّذ وبريده الإلكتروني ونوع المورد ومعرّف المورد.

### التصفية

استخدم عناصر التصفية أعلى الصفحة لتضييق النتائج:

- **الإجراء** -- تصفية حسب عمليات الإنشاء أو التحديث أو الحذف فقط.
- **المورد** -- تصفية حسب نوع مورد محدد (عميل، خدمة، شريك، مستخدم، إعدادات، نوع خدمة).
- **من / إلى** -- تصفية حسب نطاق تاريخي.

تُطبّق المرشحات فوراً عند تغييرها.

### عرض التفاصيل

انقر أي صف لتوسيعه. يعرض العرض الموسّع حمولة JSON الكاملة للإجراء المسجّل، منسّقة للقراءة. يشمل هذا قيم الحقول التي كُتبت. إذا لم تُسجّل تفاصيل لإدخال ما، تظهر رسالة "لا تتوفر تفاصيل".

السجل مقسّم إلى صفحات بمعدل 20 إدخالاً في الصفحة.


## 7. أدوات سطر الأوامر

يأتي فنك مع نصّين برمجيين لسطر الأوامر لمهام إدارية يسهل تنفيذها خارج المتصفح، مثل استرداد حساب مسؤول مغلق.

يتطلب كلا النصين أن يكون `DATABASE_URL` معيّناً في بيئتك أو في ملف `.env` في جذر المشروع.

### إعادة تعيين كلمة مرور مستخدم

```bash
npm run reset-password user@example.com
```

يولّد هذا كلمة مرور مؤقتة عشوائية من 16 حرفاً، ويحدّث تجزئة كلمة المرور في قاعدة البيانات، ويضع علامة `firstLogin = true` على الحساب ليضطر المستخدم لتغيير كلمة المرور عند تسجيل الدخول التالي.

لتعيين كلمة مرور محددة بدلاً من عشوائية:

```bash
npm run reset-password user@example.com -- --password MyNewPassword123!
```

> **تحذير أمني:** استخدام علامة `--password` ينطوي على عدة مخاطر. تكون كلمة المرور مرئية في سجل الطرفية ولأي شخص لديه وصول إلى شاشة المسؤول. كما تتجاوز إجبار `firstLogin`، مما يعني أن المستخدم لن يُجبر على تغيير كلمة المرور عند تسجيل الدخول التالي. يسهّل هذا الخيار أيضاً تعيين كلمات مرور ضعيفة أو مشتركة. استخدمه فقط في حالات الاسترداد الطارئة، وأبلغ المستخدم فوراً بتغيير كلمة مروره بعد تسجيل الدخول.

عند تقديم كلمة مرور محددة، لا تُضبط علامة `firstLogin`، ولا يُجبر المستخدم على تغييرها.

يطبع النص كلمة المرور الجديدة في الطرفية. خزّنها بأمان أو شاركها مع المستخدم عبر قناة مشفّرة فوراً.

### عرض جميع المستخدمين

```bash
npm run list-users
```

يطبع جدولاً بجميع حسابات المستخدمين مع أسمائهم وبريدهم الإلكتروني ودورهم وحالتهم (نشط أو قيد الانتظار) وتاريخ الإنشاء. "قيد الانتظار" تعني أن المستخدم لم يكمل بعد تغيير كلمة المرور عند تسجيل الدخول الأول.

### متى تستخدم أدوات سطر الأوامر

استخدم أدوات سطر الأوامر عندما:

- يكون حساب المسؤول مغلقاً ولا يمكنك الوصول إلى واجهة المتصفح.
- تحتاج لمراجعة سريعة لمن لديه وصول قبل منح مستخدم دور المسؤول.
- تكتب سكريبت نشر وتحتاج للتحقق من اكتمال الإعداد بشكل صحيح.

لإعادة تعيين كلمات المرور وإدارة المستخدمين اليومية، استخدم واجهة المتصفح في الإدارة > المستخدمون.


## 8. الأدوار والصلاحيات

لدى فنك ثلاثة أدوار. عيّن الدور الأقل صلاحيات الذي يلبي احتياجات المستخدم.

| الإجراء | مسؤول | محرر | مشاهد |
|---------|:-----:|:----:|:-----:|
| عرض العملاء والشركاء | نعم | نعم | نعم |
| إنشاء وتعديل العملاء والشركاء | نعم | نعم | لا |
| حذف العملاء والشركاء | نعم | لا | لا |
| إدارة الخدمات (إنشاء، تعديل، حذف) | نعم | نعم | لا |
| إدارة المستخدمين | نعم | لا | لا |
| إدارة الإعدادات | نعم | لا | لا |
| عرض سجل المراجعة | نعم | لا | لا |
| تصدير البيانات | نعم | لا | لا |

**مسؤول** -- وصول كامل لجميع الميزات بما في ذلك إدارة المستخدمين والإعدادات وسجل المراجعة. عيّنه للأشخاص المسؤولين عن تشغيل وتهيئة نسخة فنك.

**محرر** -- يمكنه إنشاء وتعديل العملاء والشركاء وخدماتهم، لكن لا يمكنه حذف العملاء أو الشركاء، وليس لديه وصول للصفحات الإدارية. عيّنه لموظفي العمليات الذين يديرون بيانات العملاء يومياً.

**مشاهد** -- وصول للقراءة فقط للعملاء والشركاء. لا يمكنه إنشاء أو تعديل أو حذف أي سجلات. عيّنه لأصحاب المصلحة الذين يحتاجون رؤية بيانات العملاء دون القدرة على تغييرها.
